# 原型范围说明（Prototype Scope）

## 1. 项目定位
青春智慧运动健康系统是面向青年用户的 Web 应用原型，聚焦“健康数据管理 + 运动饮食管理 + 社交激励 + 智能建议”闭环。  
当前文档用于指导毕业设计原型开发，不涉及微服务拆分。

## 2. 目标用户
- 在校大学生与青年上班族
- 有日常运动与饮食管理需求的用户
- 需要通过可视化和社交方式提升健康行为持续性的用户

## 3. 业务边界
### 3.1 纳入范围
1. 用户与权限管理（JWT + RBAC）
2. 用户健康档案与体征记录
3. 运动记录管理
4. 饮食记录与食物库
5. 统计分析与图表可视化
6. 个性化建议（规则引擎 + LLM 预留接口）
7. 周报/月报生成
8. 基础社交（打卡、目标、发帖、评论、点赞）
9. 第三方运动数据同步（mock provider，增量同步，幂等去重）

### 3.2 不纳入范围
- 医疗诊断与处方建议
- 真实第三方平台正式授权接入（仅实现 mock）
- 多租户 SaaS 场景
- 微服务与分布式事务治理

## 4. 非功能性要求
- 前后端分离，分层单体架构
- RESTful API，统一返回格式
- 统一异常处理与参数校验
- OpenAPI 自动化文档
- MySQL 8 持久化，Redis 7 缓存与会话/热点数据支持
- Flyway 管理数据库版本迁移
- 后端 JUnit 测试；前端至少保证 `npm run build` 通过

## 5. 角色定义
- 普通用户：维护个人数据、查看建议、参与社交
- 管理员：管理用户、角色权限、食物库、规则、运营内容
- 系统任务：报表生成、第三方同步、统计汇总

## 6. 阶段交付原则
- 每轮仅交付一个阶段
- 阶段结束需包含：已完成内容、运行命令、验证方式、下一轮建议
- 除非明确要求，不引入微服务

## 7. 开发计划与里程碑（9 阶段）
1. 阶段 1：工程初始化与基础规范  
里程碑：后端与前端脚手架建立，统一响应/异常/校验，JWT 基础链路与 OpenAPI 可访问。
2. 阶段 2：用户与权限管理  
里程碑：注册登录、用户管理、角色权限配置、接口鉴权生效。
3. 阶段 3：健康档案  
里程碑：健康档案与体征记录 CRUD、趋势查询接口打通。
4. 阶段 4：运动记录  
里程碑：运动类型与运动记录管理完成，日/月统计可查询。
5. 阶段 5：饮食记录与食物库  
里程碑：食物库、饮食记录、营养汇总统计完成。
6. 阶段 6：统计分析与可视化  
里程碑：核心分析接口完成，前端仪表盘图表接入。
7. 阶段 7：个性化建议与报表  
里程碑：规则建议、周报/月报生成与查询完成，LLM 接口占位。
8. 阶段 8：基础社交  
里程碑：目标、打卡、发帖、评论、点赞闭环完成。
9. 阶段 9：第三方数据同步与收尾  
里程碑：mock provider 增量同步、幂等去重、任务日志与最终联调测试完成。
